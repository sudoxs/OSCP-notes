## **مقایسه جامع ساختار کیف پول: Bitcoin vs Monero**

| مولفه | در Bitcoin | در Monero | **هدف اصلی** | **مثال فرضی (غیرواقعی برای درک)** |
| :--- | :--- | :--- | :--- | :--- |
| **1. SEED / عبارت بازیابی** | **Seed Phrase** (معمولا ۱۲ یا ۲۴ کلمه). از آن یک "Master Private Key" ساخته میشود. | **Mnemonic Seed** (معمولا ۱۲، ۱۳ یا ۲۵ کلمه). از آن "Master Private Key" ساخته میشود. | **بازیابی کامل کیف پول**. تنها چیزی که باید امن ذخیره شود. | `apple banana cherry ... zone` (این ۱۲ کلمه کل گاوصندوق شماست) |
| **2. MASTER PRIVATE KEY** | از Seed مشتق میشود. یک کلید ۲۵۶ بیتی. ریشه درخت کلیدها (HD Wallet). | از Seed مشتق میشود. یک کلید ۲۵۶ بیتی. ریشه همه کلیدهای خصوصی. | تولید تمام کلیدهای خصوصی بعدی به صورت سلسله مراتبی. | `0C28FCA3...` (یک عدد بسیار بزرگ که از Seed مشتق شده) |
| **3. PRIVATE KEYS (کلیدهای خصوصی)** | **کلید خصوصی اصلی** (از Master Key مشتق میشود). برای هر آدرس یک کلید خصوصی منحصر به فرد وجود دارد (در HD Walletها به صورت deterministically تولید میشوند). | **دو کلید خصوصی مجزا:**<br>• **Private Spend Key** (a)<br>• **Private View Key** (b)<br>هر دو از Master Key مشتق میشوند. | **Bitcoin:** امضای تراکنش برای خرج کردن.<br>**Monero:** <br>• **a:** برای خرج کردن و امضا.<br>• **b:** برای رمزگشایی و مشاهده تراکنشهای ورودی. | **BTC:** کلید درب خانه شماره ۱.<br>**XMR:** <br>• **a:** کلید گاوصندوق.<br>• **b:** کلید پنجره شیشهای یکطرفه (میتوانی ببینی داخل، اما نمیتوانی وارد شوی). |
| **4. PUBLIC KEYS (کلیدهای عمومی)** | **کلید عمومی** از کلید خصوصی با عملیات بیضوی (ECDSA) تولید میشود. هر آدرس BTC از یک کلید عمومی ساخته میشود. | **دو کلید عمومی مجزا:**<br>• **Public Spend Key** (A) از a ساخته میشود.<br>• **Public View Key** (B) از b ساخته میشود. | **Bitcoin:** تولید آدرس برای دریافت.<br>**Monero:** ترکیب A و B برای ساخت آدرس اصلی. | **BTC:** قفل درب خانه شماره ۱.<br>**XMR:** <br>• **A:** قفل گاوصندوق.<br>• **B:** قفل پنجره شیشهای. |
| **5. ADDRESS (آدرس)** | **آدرس عمومی** (مثل `1A1zP1...` یا `bc1q...`). از هش گرفتن کلید عمومی و افزودن checksum ساخته میشود. **ثابت و قابل رهگیری**. | **آدرس عمومی استاندارد** (مثل `4A1zP1...` یا `8A1zP1...`). ترکیب رمزگذاری شده **A + B + checksum**. اما... | **Bitcoin:** شناسه عمومی برای دریافت پرداخت. همه تراکنشهای این آدرس روی بلاکچین آشکار است.<br>**Monero:** شناسه عمومی **فقط برای اولین بار**. | **BTC:** "خیابان اصلی، پلاک ۱۰۰". همه میدانند چه کسی در آن زندگی میکند و چه رفتوآمدهایی دارد.<br>**XMR:** "کد مخفی ۴۵۲-XZ". فقط برای شروع ارتباط. |
| **6. ADDRESS یکبارمصرف** | وجود ندارد. آدرس ثابت است. | **Stealth Address (آدرس پنهان)**. **برای هر تراکنش ورودی جدید به صورت خودکار و یکتا** توسط فرستنده از روی آدرس اصلی شما ساخته میشود. | **حریم خصوصی**. هر تراکنش دریافتی به یک آدرس منحصربهفرد (مشتقشده از کلیدهای عمومی شما) میرود. فقط شما با Private View Key میتوانید آنها را به خودتان مرتبط کنید. | **XMR:** فرستنده با آدرس اصلی شما (`4A1zP1...`) یک "آدرس یکبارمصرف" جدید میسازد، مثلاً `9B2yR3...`. پول به این آدرس میرود. برای ناظر بیرونی، `9B2yR3...` هیچ ربطی به `4A1zP1...` ندارد. |
| **7. نحوه دریافت پول** | فرستنده به **آدرس ثابت** شما واریز میکند. (مثل `1A1zP1...`) | فرستنده از **آدرس اصلی** شما، یک **آدرس یکبارمصرف (Stealth Address)** جدید میسازد و به آن واریز میکند. | **BTC:** مستقیم به آدرس آشکار.<br>**XMR:** غیرمستقیم به یک آدرس پنهان که فقط گیرنده میتواند آن را پیدا کند. | **BTC:** پستچی نامه را مستقیماً به پلاک ۱۰۰ میبرد.<br>**XMR:** پستچی نامه را به یک صندوق امن مخفی میبرد که فقط شما آدرس آن را میدانید. او حتی نمیداند آن صندوق متعلق به صاحب کد "۴۵۲-XZ" است. |
| **8. نحوه مشاهده تراکنشها** | هرکسی میتواند با جستجوی آدرس در اکسپلورر (مثل blockchain.com) **تمام تاریخچه تراکنشها و موجودی** آن آدرس را ببیند. | **فقط دارنده Private View Key** میتواند با اسکن بلاکچین، تراکنشهای ورودی به آدرسهای یکبارمصرف خود را پیدا و رمزگشایی کند. | **BTC:** شفافیت کامل.<br>**XMR:** حریم خصوصی کامل برای گیرنده. | **BTC:** همه از پشت حصار خانه شما میتوانند داخل را ببینند و بشمارند چند ماشین در گاراژ است.<br>**XMR:** خانه شما دیوارهای بلند و پنجرههای یکطرفه دارد. فقط شما از داخل میتوانید ببینید چه چیزهایی وارد شده. |
| **9. نحوه خرج کردن (امضا)** | با **کلید خصوصی** مربوط به همان آدرس، تراکنش امضا میشود. | با **Private Spend Key (a)** و با کمک **Private View Key (b)** برای یافتن خروجیهای خود، تراکنش امضا میشود. | **BTC:** اثبات مالکیت خروجیهای یک آدرس خاص.<br>**XMR:** اثبات مالکیت خروجیهای پنهان، بدون فاش کردن کدام خروجی متعلق به شماست. | **BTC:** با کلید درب خانه ۱۰۰، ماشین را از گاراژ همان خانه خارج میکنی.<br>**XMR:** با کلید گاوصندوق، بدون اینکه کسی بداند از کدام صندوق مخفی پول برداشتی، آن را خرج میکنی. |

## **مثال عملی و داستانی کامل برای درک بهتر**

### **سناریو: علی میخواهد از سارا ۰.۱ واحد دریافت کند.**

#### **در شبکه Bitcoin (BTC):**
1.  **علی** به **سارا** میگوید: "لطفاً به آدرس من `1AliBTCAddress123456789` واریز کن."
2.  **سارا** یک تراکنش ایجاد میکند: "از آدرس من به آدرس `1AliBTCAddress123456789`، ۰.۱ بیتکوین بفرست."
3.  تراکنش در بلاکچین ثبت میشود.
4.  حالا **هر شخصی در دنیا** میتواند:
    *   به سایت blockchain.com برود.
    *   آدرس `1AliBTCAddress123456789` را سرچ کند.
    *   ببیند که در تاریخ X، ۰.۱ BTC دریافت کرده است.
    *   موجودی کنونی این آدرس را ببیند.
    *   تمام تراکنشهای گذشته و آینده این آدرس را رهگیری کند.

#### **در شبکه Monero (XMR):**
1.  **علی** به **سارا** میگوید: "لطفاً به آدرس مونرو من `4AliXMRMainAddressAbCdEfGhIjKlMnOp` واریز کن."
2.  **سارا** کیف پولش را باز میکند و آدرس علی را وارد میکند.
3.  **کیف پول سارا به صورت خودکار و پشت صحنه:**
    *   از روی **Public Spend Key (A)** و **Public View Key (B)** موجود در آدرس علی، یک **آدرس یکبارمصرف (Stealth Address) جدید و منحصربهفرد** میسازد. مثلاً `9StealthForThisOneTxXyZwVuTsRq`.
    *   پول را به این آدرس جدید میفرستد.
    *   همچنین یک **رمز یکبارمصرف** میسازد و آن را در تراکنش پنهان میکند.
4.  تراکنش در بلاکچین ثبت میشود. برای **هر ناظر بیرونی**:
    *   آدرس مقصد در تراکنش **`9StealthForThisOneTxXyZwVuTsRq`** است.
    *   این آدرس **هیچ شباهت آشکاری** به آدرس اصلی علی (`4AliXMRMainAddress...`) ندارد.
    *   نمیتوان فهمید این پول برای علی است.
5.  **چگونه علی پول خود را میبیند و خرج میکند؟**
    *   کیف پول علی بهطور مداوم بلاکچین را اسکن میکند.
    *   او **Private View Key** خود را دارد. با این کلید میتواند آن "رمز یکبارمصرف" پنهان در تراکنش را آزمایش کند.
    *   وقتی رمز با کلیدش مطابقت کرد، میفهمد که این تراکنش یکی از "آدرسهای یکبارمصرف" متعلق به اوست و آن را به موجودی خود اضافه میکند.
    *   وقتی بخواهد خرج کند، با **Private Spend Key** خود امضا میکند، بدون اینکه فاش شود کدام یک از آن خروجیهای پنهان متعلق به او بوده است.

### **جمعبندی نهایی در یک نگاه:**

| ویژگی | Bitcoin | Monero |
| :--- | :--- | :--- |
| **شفافیت** | کامل. همه چیز عمومی است. | صفر. همه چیز خصوصی است. |
| **ارتباط آدرسها** | همه تراکنشهای یک آدرس بهراحتی قابل پیوند هستند. | تراکنشها غیرقابل پیوند هستند. هر دریافت یک آدرس جدید میسازد. |
| **موجودی عمومی** | قابل مشاهده توسط همه. | فقط برای مالک قابل مشاهده است. |
| **کلیدها** | یک جفت کلید (خصوصی/عمومی) در هر آدرس. | دو جفت کلید مستقل (مشاهده/خرج) برای هر کیف پول. |
| **آدرس** | ثابت و قابل استفاده مکرر (البته نه推薦). | اصلی ثابت است، اما برای هر تراکنش یک آدرس پنهان جدید ایجاد میشود. |
| **هدف فلسفی** | شفافیت، عدم نیاز به اعتماد، پول دیجیتال غیرمتمرکز. | حریم خصوصی، جانشینی پول نقد، غیرقابل رهگیری. |

**پس به زبان بسیار ساده:**
*   **بیتکوین مثل یک دفترکل عمومی است** که نام و تراکنشهای همه در آن نوشته شده.
*   **مونرو مثل یک شبکه مخفی است** که هر کس یک نام مستعار دارد و برای هر معامله از یک نام مستعار کاملاً جدید استفاده میکند، و فقط طرفین معامله میدانند که پول بین آنها جابهجا شده است.

## **⚠️ بخش‌های نیاز به اصلاح:**

### **۱. مشکل: "Public Key مشتری رو بگیرم"**
```bash
# اشتباه: Public Key نمی‌گیری!
# درست: آدرس Bitcoin می‌گیری

# فرقشون:
Public Key = مثل شماره شناسنامه (برای امضا)
آدرس = مثل شماره حساب بانکی (برای دریافت پول)

# پس:
مشتری باید بهت بده: "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa" (آدرس)
نه: "04b5b5..." (Public Key)
```

### **۲. مشکل: "میکرو SD رو به سیستم آنلاین می‌زنم"**
```bash
# خطرناک! اینطوری:
سیستم آنلاین ← میکرو SD ← تراکنش امضا شده ← broadcast

# درست:
سیستم آفلاین ← میکرو SD ← تراکنش امضا شده
↓
سیستم آنلاین (مثلاً Tails با Tor) ← تراکنش رو وارد کن ← broadcast
# میکرو SD رو مستقیم به سیستم آنلاین نزن!
```

### **۳. مشکل: "Public Key من بدم"**
```bash
# برای Monero در Feather:
Feather آدرس می‌ده، نه Public Key
آدرس Monero: "۸۷t۵iU۴pRnHAqRZFg۸h۴tgG۵۷rRLZ۶GJ۷cZ۴B۲a۵o۷E۹aX۳cV"
```

### **۴. مشکل: "از Electrum مستقیم به ویزا کارت"**
```bash
# نمیشه مستقیم!
Bitcoin → صرافی متمرکز (مثلاً Coinbase) → بانک → کارت
# یا:
Bitcoin → کارت بیت‌کوینی (مثل BitPay) → خرید
```
