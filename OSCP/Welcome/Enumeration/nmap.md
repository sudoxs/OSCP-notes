| فلگ / آپشن                  | معنی / کارکرد کوتاه                                                                                                                                |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-p-`                       | اسکن **همهٔ پورت‌های TCP** (۱ تا ۶۵۵۳۵)                                                                                                            |
| `-p xx`                     | اسکن **فقط پورت(های) مشخص** (مثلاً `-p 80` یا `-p 80,443`)                                                                                         |
| `--min-rate=1000`           | حداقل نرخ ارسال پروب‌ها: تلاش برای **حداقل ۱۰۰۰ بسته در ثانیه** (سریع‌تر، ولی پرریسک‌تر برای تشخیص/از دست رفتن پاسخ‌ها)                            |
| `-sS`                       | اسکن TCP **SYN (Half-open)**؛ سریع و رایج، معمولاً نیازمند sudo/روت                                                                                |
| `-sT`                       | اسکن TCP **Connect()**؛ بدون روت هم کار می‌کند (کامل‌تر ولی کندتر/پرلاگ‌تر)                                                                        |
| `-sU`                       | اسکن **UDP** (کندتر و سخت‌تر برای تشخیص درست)                                                                                                      |
| `-sV`                       | تشخیص **نسخهٔ سرویس** (Service/Version Detection) روی پورت‌های باز                                                                                 |
| `-sC`                       | اجرای اسکریپت‌های پیش‌فرض NSE (معادل `--script=default`)                                                                                           |
| `-O`                        | تشخیص **سیستم‌عامل** با OS Fingerprinting (معمولاً نیازمند sudo)                                                                                   |
| `-A`                        | حالت “Aggressive”: ترکیبی از **`-sV` + `-O` + `-sC` + traceroute** (اسکن سنگین و پر سر و صدا)                                                      |
| `-Pn`                       | فرض کن میزبان **آنلاین است**؛ مرحلهٔ ping discovery را رد می‌کند (برای وقتی ICMP/پینگ بلاک است)                                                    |
| `-sn`                       | فقط **Host discovery** (پینگ/تشخیص زنده بودن)، **بدون اسکن پورت** (قدیمی: `-sP`)                                                                   |
| `-oG x.txt`                 | خروجی در قالب **Grepable** داخل فایل (قدیمی/Deprecated ولی هنوز دیده می‌شود)                                                                       |
| `-v`                        | خروجی **Verbose**؛ اطلاعات بیشتر حین اجرا                                                                                                          |
| `-oN file`                  | (در مثال شما نیست، ولی رایج) خروجی نرمال به فایل                                                                                                   |
| `-oX file`                  | (رایج) خروجی XML به فایل                                                                                                                           |
| `--osscan-guess`            | وقتی OS دقیق نیست، **حدس‌های تهاجمی‌تر** برای سیستم‌عامل می‌دهد                                                                                    |
| `--script http-headers`     | اجرای اسکریپت NSE برای گرفتن **HTTP Headerها**                                                                                                     |
| `--script "vuln"`           | اجرای دسته اسکریپت‌های NSE با ردهٔ **vuln** (بررسی آسیب‌پذیری‌های شناخته‌شده)                                                                      |
| `--script "cve.x.x.x"`      | اجرای یک اسکریپت NSE مشخص (اسم اسکریپت/الگو)                                                                                                       |
| `--script-updatedb`         | آپدیت کردن دیتابیس اسکریپت‌های NSE بعد از افزودن/تغییر اسکریپت‌ها                                                                                  |
| `--script smb-os-discovery` | اسکریپت NSE برای **تشخیص OS/اطلاعات SMB** (معمولاً روی 139/445)                                                                                    |
| `--script http-enum`        | اسکریپت NSE برای **Enumerate کردن مسیرها/اپ‌های وب رایج** روی HTTP                                                                                 |
| `-z`                        | **Zero-I/O mode** فقط برای **Ncat** است، نه nmap. معنی: هیچ داده‌ای نفرست، فقط اتصال/باز بودن را تست کن. (اگر منظورت nmap است، معادل مستقیم ندارد) |
| `-w 1`                      | این هم معمولاً برای **Ncat/Netcat** است (timeout). در nmap به شکل دقیق این نیست؛ nmap گزینه‌های زمان‌بندی جدا دارد.                                |
| `x.x.x.3-251`               | رنج IP از `.3` تا `.251`                                                                                                                           |
| `x.x.x.x/24`                | نت‌ورک /24 یعنی کل رنج **۲۵۶ آدرس** (معمولاً `.1` تا `.254` قابل استفاده)                                                                          |
